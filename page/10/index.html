谷歌将于下周选择部分企业加入Google+测试

Posted on July 16, 2011

据国外媒体报道，谷歌将加快Google+社交服务在企业领域的部署，计划于下周选择哪些企业参加Google+企业账户测试，并将在随后公布这些企业名称。谷歌产品经理克里斯蒂安·奥斯蒂利安（Christian Oestlien）周四通过Google+博客表示，大量企业申请加入Google+，因此谷歌加快Google+企业账户测试进程。

不过要想让自己的企业、品牌、学校或其它机构名称加入Google+，那么应当迅速行动，因为奥斯蒂利安透露，谷歌将于周五停止接受申请。

奥斯蒂利安表示：“对于成千上万提出Google+申请的企业，我们表示感谢！我们无法接纳所有企业，但你们表现出的极大兴趣促使我们加快了部署计划。”

上周，奥斯蒂利安表示，谷歌计划在今年晚些时候推出Google+企业账户。不过他没有透露加快计划后推出Google+企业账户的具体时间。

随着越来越多的企业希望进入极具活动且发展迅猛的社交网络领域，企业账户的重要性也日益明显。Facebook和Twitter这两大社交网络均提供企业账户服务。

如果一家企业能够在社交网络上通过好友不断传播视频介绍，那么这家企业的知名度一定会有提升。更重要的是，从长远来看，企业通过社交网络可以在客户支持、好友推荐，以及企业与客户交流方面受益更多。

不过，谷歌的企业版Google+社交服务尚未完全准备就绪，有不少企业在目前版本的Google+服务中建立了账户，但被谷歌删除。不过也有一些企业成为幸运儿，例如汽车品牌福特的Google+账户就一直存在，即便谷歌尚未公布企业版Google+的测试名单。

奥斯蒂利安表示，包括他自己，以及Google+企业资料产品经理丹尼斯·特罗佩尔（Dennis Troper）在内的整个Google+团队“下周将专注于选择一些企业伙伴参加测试，我们将在下周与这些伙伴进行交流，并将在随后公布这些企业的名单”。

不过，一些具有个人品牌效应的名人建立Google+账户则非常方便，例如美国男演员威尔·惠顿（Wil Wheaton）、女演员菲利西亚·戴（Felicia Day）、女演员艾丽莎·米拉诺（Alyssa Milano），以及说唱歌手50 Cent。

Damn Small Linux让旧电脑焕然一新

Posted on July 14, 2011

新闻来源:solidot.org
Osnews的一篇文章介绍了发行版Damn Small Linux如何让旧电脑恢复青春。 现今的应用程序绝大多数不可能在再运行在Pentium II、I或486等处理器上，十多年前的操作系统Windows 3.1、95、98、98SE、ME和2000也不再安全，微软已经停止了安全更新，也鲜有杀毒软件能支持这些过时的操作系统。

Damn Small Linux在 2003年首次面世时，其目的便是为了创造能运行在旧电脑上的Linux发行版，它禁用了所有不必要的守护程序和服务，可使用旧电脑支持的任何存储设备， 甚至可作为客户机操作系统在Windows中运行。它最新版本是2008年发布的DSL 4.4.10，运行2.4 Linux kernel；还有一个姐妹项目DSL-N，运行2.6 Linux kernel，硬件最小需求比DSL高，至少需要64 M内存和200 mhz奔腾处理器。该发行版已经停止了更新，原因是作者将其他开发者都踢了出去，原开发者创建了新的发行版Tiny Core Linux。

Linux应该由谁来领导?

Posted on June 27, 2011

新闻来源:51CTO
Linux表面上非常风光，但背后它面临的问题太多了，应该引起高度关注，当然我指的不是Unity中难以捉摸的启动器，也不是GNOME 3中缺失的最小化按钮，而是人，截至目前，还没有一个人对Linux起着绝对领导作用。

假如有一天，让你选择一个人来领导Linux，你要选择谁?如果你一时半会想不到合适的人选，我倒是可以推荐几个。

Richard Stallman

资格/能力

Richard Stallman(全名Richard Matthew Stallman，又被称为RMS)是终极极客一枚，经常留着一把长长的胡子，Richard Stallman出生在纽约，毕业于麻省理工学院，除了对C，C++，Java和elisp能出口成章外，他的法语和西班牙语也非常棒。

他创建了著名的文本编辑器Emacs，一个有时被当作操作系统使用的文本编辑器，虽然他的编码技术众所周知，但另他真正脱颖而出的是，他对培育自由软件生态系统所做的大量工作。

老实说，如果这个世界上没有Richard Stallman，我们可能会从1000家不同的公司购买软件，他是第一位真正关心用户自由而不是开发商钱袋子的人，他还开发了 copy left，Copyleft是一种许可形式，用于维护计算机，软件，文档和艺术的版权，这还不是全部，他还利用他的名望反映各种社会问题。

弱点

RMS是FOSS理念的坚定信仰者，任何情况下，他的价值观都不会妥协，但这却成了他的一个弱点。正如Shuttleworth在UDS会议期间提出的，为了Linux的未来，必须要有一个舵手来掌管航行的方向，并且要做出一些妥协，哪怕是和我们的核心价值有冲突，而RMS基本上是不可能愿意这么干的。

他如何帮助/领导Linux?

事实上，他已经在许多方面帮助Linux发展，但要让他走到台前来领导似乎不太可能，RMS喜欢悠闲，朴实的生活，而且他的年纪也不小了，在这个时候出来承担如此重任，的确有点为难他。他的重点也不同，他更关注的是Copyleft，GNU和政治活动，因此，即使他想，他也不能帮助Linux，话虽如此，但如果能得到他的援手，Linux也会受益良多，例如，他在全世界宣扬使用专有软件的陷阱，我们希望他能继续这项工作，不要停下来。

Linus Torvalds

资格

有关Linus Torvalds最酷的话题要数他的硕士论文了，他的论文题为“Linux：一个可移植的操作系统”。Linus Torvalds出生于芬兰赫尔辛基，他的名字来源于获得诺贝尔奖的化学家Linus Pauling，他还拥有芬兰军队少尉军衔，他在军队是一名弹道计算人员。他最大和最流行的成就就是Linux，全世界最好的操作系统。他曾经编写了 Pac Man(吃豆人)的克隆版本Cool Man，和RMS一样，Linus深受黑客，程序员和Linux爱好者喜欢。他的名字还用来命名了一颗小行星。

弱点

和RMS一样，Linus也喜欢悠闲的生活，你无法想象让他来担任史蒂夫乔布斯或鲍尔默的角色会是什么样子，他的全部精力几乎都倾注到 Linux内核的改进和发布。最初，他是一名KDE粉丝，在KDE 4后他转向了GNOME，因此，他不会对任何特定项目调整自己，即使是目前很火的Android。

他如何帮助/领导Linux?

Linus帮助Linux最好的方法是继续为内核做贡献，当新设备，如平板电脑风靡市场时，他将要对Linux内核的未来作出重大决策，他需要做的就是确保Linux内核保持稳定和流行。

Mark Shuttleworth

资格

这个人已经看到了世界，我指的是从外太空看世界，他是世界上第二位自费太空游客。1995年，他创立了Thawte安全认证，然后以5.75亿美元的价格卖给了Verisign，随后，他成为了一名Debian操作系统开发人员，到了2004年，他开始了Ubuntu的开发，并投资10亿美元成立了 Ubuntu基金会，从那时起，他就成为Linux爱好者忠心拥护的大神。

弱点

虽然在Mark Shuttleworth管理下，Ubuntu一跃成为最流行的Linux发行版，但他也做了一些损害Linux社区的决策，无论是转移到Unity，还是将Ubuntu按钮移到窗口左边，总是有很多人不赞成Shuttleworth的做法。此外，也有一些人认为，如果让他在前面领导Linux，尚缺乏经验，要赢回这些人的心，只有靠未来问世的Ubuntu了。

他能怎么帮助Linux?

他帮助Linux的最好办法就是确保Ubuntu成为最好的发行版，这将促进Linux社区的良性竞争，有助于促使 Fedora，OpenSUSE 等发行版加快创新的步伐。实际上，Shuttleworth还是KDE基金会的第一位赞助人，我们对他的所有期望是，希望他能继续帮助整个Linux社区。

谁应该是Linux的最终领导者?

在分析了这几位对Linux做出重大贡献的人士后，“百万美元”问题的答案仍然未揭晓 — 谁应该领导Linux?他/她能像乔布斯那样把冰箱销售给爱斯基摩人吗?或者，他/她能像施密特那样有大量的管理经验吗?

也许这个问题根本就没有任何意义，除非Linux先解决另一个主要问题，即Linux究竟需要什么?换句话说就是，如何才能将Linux社区汇聚到一起?如果可以，它会以什么形式汇聚呢?

截至目前，Linux社区还处于一种无政府状态，要达到微软、Google和苹果多年来早已实现的目标，Linux必须尽快从无政府状态过渡到民主体制，或者善意的独裁更好，这是一个长期的辩论话题，你觉得是善意的独裁还是民主更好?

如果你使用Ubuntu的时间足够长，你一定注意到它在Shuttleworth的领导下，已经逐步从无政府状态过渡到善意的独裁，Linux 社区是一个尚未开发的潜力巨大的储备资源，实际上，许多在Linux上工作的开发人员都是黑客，而且才华横溢，唯一剩下要做的就是将他们组织到一起，激励他们朝一个共同的目标迈进，如果能出现这样一个人，有这样的能力，那么他/她就应该成为Linux的领导者。

你的想法

不知道你对Linux社区缺乏领导是怎么看的?这种无政府状态对Linux社区来说是好事还是坏事?你希望由谁来领导Linux社区?是Mark Shuttleworth、Linus Torvalds、Richard Stallman还是其他人?

Linux无所不在的10个方面

Posted on June 18, 2011

新闻来源:51CTO
长期以来，Linux在服务器领域扮演领军角色，这在很大程度上得益于它具有稳定性、安全性、总体拥有成本较低等优点。不过许多人也许没有认识到，Linux在生活的其他方面其实也变得非常地普遍。

不仅像Ubuntu这些发行版在帮助Linux在桌面领域取得了巨大的进步，这款开源操作系统如今还在背后悄悄地为我们许多人习以为常的无数创新提供支持。

如今还能在哪里看到Linux的身影?下面仅仅是Linux让生活更美好的众多方面的几个典例。

1、Android
当然，首当其冲的是Android这款谷歌的基于Linux的移动操作系统。Android一心就想把苹果大获成功的iPhone挤入到小众设备领域，同时还在与大受欢迎的iPad作一番较量。Android把Linux交到了全球无数移动设备消费者的手里，大概是Linux迄今为止在主流界取得的最大成功。
之外还有Chrome操作系​​统，据说这款操作系统会在华硕今年夏天推出的一款低成本笔记本电脑中扮演重要角色，当然它还出现在其他设备中。

2、WebOS
可能与Android一样大有潜力的是惠普近期的这一决定：几乎在其所有的台式机和笔记本电脑上使用WebOS，更不用说惠普的平板电脑和手机了。当然，WebOS是一款基于Linux的操作系统，最初由Palm公司开发。有了惠普的这项新战略，Linux还会出现在无数消费者的桌面系统、便携式设备和手持装置中，这有望从另一方面极大地推动Linux进入到广泛的主流领域。

3、小众个人电脑
甚至除了已经在运行Linux的所有许多消费级个人电脑外，我们还开始看到这款操作系统也在背后为小众个人电脑提供支持。就拿最近首次亮相的KiWi个人电脑来说吧。这款装有Ubuntu的电脑面向老年人使用Linux，让老年人用户可以轻松自如地操作电脑。该设备的网站这样解释：“KiWi个人电脑采用Ubuntu操作系统，从电脑启动到关闭，该操作系统可以提供简单明了的导航浏览操作。Ubuntu操作系统建立了对用户友好的桌面环境，让老年人可以立即访问电子邮件和互联网，轻松无忧。”别扯什么Linux更难使用的鬼话了。

4、机顶盒
虽然Linux对机顶盒用户来说通常是看不见的，但它常常是非常普及的机顶盒的核心;比如说，我们许多人利用机顶盒来流式传送内容。运行Ubuntu的Neuros Link网络视频播放设备就是个例子。

5、自动柜员机(ATM)
多年来许多专家一直在警告，Windows对银行业务来说其实不够安全，所以看到Linux日益应用于自动柜员机领域也就不足为奇了。比如说，巴西南里奥格兰德银行(Banrisul)的一个Linux项目就备受瞩目。

6、车载电脑
最近宣布的另一项Linux创新就是为农用和建筑施工车辆设计的一系列个人电脑。Grayhill设备就是为恶劣条件和严酷环境而设计的，它让用户可以选择Linux或Windows CE。
Linux之前还出现在了许多车辆上，包括Mavizen屡获奖项的TTX02电动摩托车。另外别忘了这一点：包括丰田和标致在内的多家汽车厂商也在使用Linux操作系统。

7、维基百科、谷歌及更多
如果你曾经用过维基百科，那么你已经从Linux得到了好处，至少间接得益。实际上，在使用了多年的红帽和Fedora后，维基百科现在又在使用Ubuntu。可扩展性如何?维基百科现在每月的页面浏览量大约是100亿人次，据说这一切离不开Ubuntu的支持。
如果我们把话题转向企业应用，谷歌、亚马逊、思科、IBM、纽约证券交易所和维珍美国公司也都是大名鼎鼎的Linux用户。

8、每个儿童一台笔记本电脑
每个儿童一台笔记本电脑(One Laptop Per Child)项目组织旨在开发出一种电脑，以便分发给全世界成千上万处于困境的孩子;它已如愿开发出了XO笔记本电脑，可充分利用Linux在自由方面的许多优势。

9、政府
单单在美国，美国邮政管理局、美国国防部和美国联邦航空局都是Linux的重要政府用户。另外世界上还有许多政府做出了使用Linux的类似决定，包括法国、西班牙、德国和瑞士。

10、IBM的沃森
最后但并非最不重要的是，最近在美国智力竞赛节目《危险边缘》中以出众性能而让世人惊叹的沃森(Watson)超级计算机就是运行在Linux操作系统上。
当然，上述这些只是Linux的许多应用领域的几个代表。不过，它们一同相当有说服力地证明了这款开源操作系统具有的功能和魅力。难怪越来越多的公司企业摈弃传闻，切切实实地改用Linux。

新闻来源:LXer

于是 Linux 失去了潜在用户……

一步步教你安装Win7+Mac OS X Snow Leopard+Ubuntu10.10三系统

Posted on May 7, 2011

前言：折腾了几天终于把这三个系统装好了，本来昨天都想写教程的了，但是自己用了那个boot think的# b=0命令导致我任何一个系统都进入不了，我也搞不清楚是怎么回事，所以只能全部重新安装了一遍，现在终于安装并配置好这三个系统了，现在有空就写篇教程 让大家做个参考吧

我的电脑配置：
CPU：AMD 3500+
主板：映泰Tforce 550
显卡：影驰 8600GT魔灵
硬盘：希捷 酷鱼 160G
内存：威刚 万紫千红 DDR2 667 1.5G
显示器：AOC 772S
声卡：Realtek ALC861
网卡：Marwell 88E8053

电脑比较旧了，是2006年买的

前期准备工作：
1.一张win7光盘
2.一张ubuntu10.10光盘
3.Mac OS X Snow Leopard10.6.3原版镜像
4.一张winPE光盘（留着备用，以防安装了雪豹进不了win7用的）
5.安装Mac OS X Snow Leopard所需要的工具包，驱动补丁和内核补丁

1，2，4相信大家都比较容易找到，我只提供Mac OS X Snow Leopard10.6.3原版镜像的BT种子
还有安装Mac OS X Snow Leopard10.6.3所需要的工具包，驱动补丁和内核补丁

三个系统安装步骤是：win7->Mac OS X Snow Leopard->ubuntu10.10

第一步：分区
我的硬盘是160G的，我的分区如下：
C盘：30G主分区(安装win7）
D盘：30G逻辑分区（存放资料）
E盘：30G逻辑分区（存放音乐和电影）
F盘：30G逻辑分区（安装雪豹）
剩下40G不分配，留给安装ubuntu10.10

你的硬盘可能是500G或者1T的，那你就自己规划下该如何分区吧

第二步：安装win7
把win7光盘放进光驱，重启进入win7安装界面，安装win7相对很简单，我相信大家都会的

第三步：安装雪豹
安装可以参考以下几篇经典帖子：
http://bbs.pcbeta.com/thread-763656-1-1.html
http://bbs.pcbeta.com/thread-592288-1-1.html
http://bbs.pcbeta.com/thread-794981-1-2.html
http://bbs.pcbeta.com/thread-676663-1-5.html

这里我主要讲下我的大概安装过程，和上面的帖子的安装过程基本一样的
帖子后面我会给出安装雪豹的教程大合集，是PDF和WORD文档的，方便大家阅读和参考

1.缩小当前分区，为雪豹和雪豹安装盘腾出空间
win7下打开控制面板->系统和安全->管理工具->计算机管理->磁盘管理
右键单击E盘->压缩卷->输入7168M->得到一个未分配的7G的空间
然后右键选择这未分配的7G的空间->新建->新建过程中选择不格式化

右键单击F盘->选择删除
得到一个未分配的30G空间，专门为安装雪豹准备的
然后右键选择这未分配的30G的空间->新建->新建过程中选择不格式化

2.安装hfsexplorer，jre-6u18-windows-i586，Macdrive
这三个软件都在我给出的安装Mac OS X Snow Leopard所需要的工具包里
借鉴我写出的那几个经典帖子我相信大家都知道如何装了吧
这里值得注意的地方是那个可以读写苹果分区的Macdrive软件要重启后才能对苹果分区进行读写

3.用硬盘助手把Mac OS X Snow Leopard镜像写入那7G的空间
写入完后替换OSInstall和内核补丁，这个很重要，一定要替换对了
否则进不了系统安装界面

4.安装boot think
这个软件也在我给出的安装Mac OS X Snow Leopard所需要的工具包里
具体安装方法是：先把boot think解压到C盘里去，然后打开Darwin，右键选择Install.bat以管理员身份运行
安装好后把我给出的那些补丁放到boot think里去，具体的位置是：C:DarwinSystemLibrarySLExtensions
我的放了如下补丁：
AppleACPIPS2Nub.kext
AppleIntelPIIXATA.kext
ApplePS2Controller.kext
AppleVIAATA.kext
ATAPortInjector.kext
ElliottForceLegacyRTC.kext
EvOreboot.kext
fakesmc.kext
IOAHCIBlockStorageInjector.kext
IOATAFamily.kext
JMicronATA.kext
LegacyAppleRTC.kext
LegacyHDA.kext
NullCPUPowerManagement.kext
OpenHaltRestart.kext
SleepEnabler.kext
VoodooPS2Controller.kext

你可以参考下，具体的还得你自己亲自试，后面我会给出这些补丁每个都是干啥用的

5.重启安装雪豹
重启后你会看到boot think的启动项，选择进去，选择雪豹的安装盘，顺利的话你就可以进入雪豹的安装界面了
安装雪豹的过程你可以借鉴我上面提到的几篇经典帖子

这里值得注意的是安装雪豹过程中可能会卡住不动，这时你动下鼠标就可以继续进行了
还有在你安装完雪豹后，可能提示你的是安装失败，这里不用担心，其实是安装成功了
在你安装成功或者安装失败后，你都先别急着重启电脑，否则会导致你的win7启动不了，那个系统都不进不去
如果你手里有winPE的光盘，这里可以跳过，我的是没有，所以得用命令解决
你打开雪豹的实用工具->终端
输入如下命令：
diskutil list
fdisk -e /dev/rdisk0 rdisk0里的是零，不是字母O
f 1
w
系统会问你是否写入
y
quit

重启后你还不能进入雪豹，你还得进入win7替换破解内核，就是复制破解内核到安装好的雪豹系统盘里去
替换好了重启就可以进入雪豹了，进入雪豹后就是设置一些基本信息，包括创建你的用户名

第四步：安装ubuntu10.10
把ubuntu10.10光盘放入光驱，重启进入安装界面，ubuntu安装我相信大家都会的，这里我就省略了，如有不懂可以自己百度google找图文教程

第五步：为雪豹安装驱动和软件
安装驱动我不在行，多逛逛远景论坛和啃苹果论坛吧
后面我会给出安装驱动的一般步骤，安装驱动不难，难的是要找到符合你的驱动

安装软件的话，你第一次使用雪豹可能也不大知道如何安装的
后面我会给出安装软件的一般步骤
还有下载苹果软件的网站不多
这里我提供一个比较好的
那个网站的有些软件安装是需要密码的
软件下载网站：
http://soft.macx.cn/

第六步：收尾工作
安装完ubuntu10.10后，会出现win7和雪豹的启动项，我的选择雪豹的启动项是进不去的，这个还得设置，具体办法我也不知道，没试过，我是用boot think来引导雪豹的

三个系统安装完后，还记得为安装雪豹准备的7G分区吧，相信你也想回收那7G分区，否则在电脑里看到这7G空间不舒服
方法如下：
win7下打开控制面板->系统和安全->管理工具->计算机管理->磁盘管理
右键那7G分区选择删除，然后选择E盘，我的是E盘，可能你的不一样，右键选择拓展，一路next下去那7G分区就合并到E盘里了

回收这7G分区后，你安装好的雪豹就进不去了，这里还得用命令找回来：
开始-附件-右键单击命令提示符以管理员身份运行
diskpart
select disk 0
list partition
select partition X （苹果安装所在盘)
set ID=AF

此时会发生错误提示：所选卷或分区正在使用

重启电脑再次开始-附件-右键单击命令提示符以管理员身份运行
输入：
diskpart
select disk 0
list partition
select partition X （苹果安装所在盘)
set ID=AF

此时就会成功，然后就可以引导你的雪豹了

还有可能你觉得boot think的引导项的名字不好，想改掉，OK，下面教你怎么改
开始-附件-右键单击命令提示符以管理员身份运行
1.cmd下使用bcdedit /enum all命令查看系统启动管理器 boot think的ID 值
就是类似{9a1fe72e-5444-11e0-9b1f-82fac67f6781}花括号中的值，复制。

2.使用bcdedit /set {ID} Description “Mac OS X Snow Leopard”修改。
例如：bcdedit /set {9a1fe72e-5444-11e0-9b1f-82fac67f6781} Description “Mac OS X Snow Leopard”

3.重启系统，OK

第七步：如何维护这三个系统
安装简单，如何维护这三个系统还是有些技巧的
第一最容易出问题的就是win7，重装了win7后该如何找回雪豹和ubuntu呢
找回ubuntu很简单，具体办法看如下我写的帖子：
viewtopic.php?f=139&t=316505

找回雪豹的话，也简单的，只要你重装boot think，再把那些补丁放到boot think里去就可以重新引导回雪豹了

第二雪豹出问题重装了，重装的时候要记得输入我上面红字提示的命令，预防启动不了win7
这里还要提醒的是还要记得重启后你还不能进入雪豹，你还得进入win7替换破解内核，就是复制破解内核到安装好的雪豹系统盘里去，替换好了重启就可以进入雪豹了，进入雪豹后就是设置一些基本信息，包括创建你的用户名

第三ubuntu出问题重装了，这个对win7和雪豹都没影响，所以不用做任何工作就可以正常引导win7和雪豹
